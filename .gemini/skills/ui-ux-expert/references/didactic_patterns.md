# UI/UX Expert - Didactic Financial Education Patterns

## Tone of Voice: The "Pragmatic Educator"

The app should sound like a knowledgeable, calm, and non-judgmental mentor. 

### Do:
- Use encouraging and clear language.
- Explain technical terms (e.g., "Cash Flow", "Savings", "Liquidity").
- Provide actionable but neutral tips.
- Focus on mathematical facts and financial principles.

### Don't:
- Use alarmist language (e.g., "You are spending too much!").
- Make prescriptive life choices (e.g., "Buy a car", "Stop eating out").
- Use overly complex financial jargon without explanation.

## The "Definition + Practical Tip" Pattern

Every major feature or information group should follow this pattern for its hints:

1. **The Definition:** Explain *what* the data represents in simple terms.
   - *Example:* "Needs (50%) represent your vital survival expenses like housing and food."
2. **The Practical Tip:** Provide a generic, neutral advice on how to manage that specific area.
   - *Example:* "TIP: If this group exceeds 50%, try reviewing fixed contracts like internet or insurance."

## Information Locality

- Provide information exactly where the user needs it. 
- Use "on-demand" hints (info icons) to keep the UI clean for advanced users while supporting beginners.
- Ensure that the explanation for a calculation (e.g., Balance) is next to the result.

## Visual Hierarchy for Finances (BudgetFlow Palette)

- **Income (Positive):** Use the Primary Green palette. Use upward-pointing indicators.
- **Expenses/Spent (Negative):** Use the Error Red palette. Use downward-pointing indicators.
- **Goals/Limits:** Use neutral tertiary or surface-variant colors for progress tracks to avoid visual noise.
- **Privacy:** Sensitive financial data must always have a visibility toggle option.
